act live demo an event-sourced order pipeline — place → pay → ship → deliver — instrumented with @rotorsoft/act-otel

what you're looking at

Every button commits real events to an event store, and each hop of the order lifecycle is a reaction on its own lane: payments charge placed orders, fulfillment ships paid ones, and delivery is deliberately flaky — it drops a fifth of its attempts and retries with backoff until the courier shows up. The tiles are projections: read models rebuilt from the event stream, pushed to this page over SSE the moment events land. Grafana watches the same system from the outside through the act-otel Prometheus metrics — keep it open beside this page and every click shows up in both worlds.

the poison story

A poison message is an event its reaction can never process — here, an order whose payment is rejected every time. Act retries it (attempts below), then quarantines only that stream; every other order keeps flowing. Unblocking alone would just re-block — the event redelivers and fails again — so the green button does the real operator move: fix the root cause, then release. The stuck order then pays, ships, and delivers.
