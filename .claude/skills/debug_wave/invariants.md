# Debug Wave ‚Äî invariants, ruled-out facts, and the bug log

The "survive the domain model" gate depends on knowing what is *correct by design*. A hunter that flags a design choice as a bug wastes a wave; a hunter that dismisses a real bug as "probably intended" misses it. This file is the shared memory that keeps both from happening. Read it before a wave; append to it after.

## Standing invariants a hunter must not violate

These are load-bearing facts about the framework. A red test that only "works" by contradicting one of these is testing a fantasy, not a bug.

- **Global event ids are strictly monotonic.** A later commit always gets a higher `id` than any earlier one. Any argument that depends on a future commit landing at a *lower* id ‚Äî "the scan skips it", "the watermark advances past it" ‚Äî is invalid. This single fact killed the #1254 false positive: advancing an empty-fetch stream to a window-max can never skip a future event, because a future event will have a higher id than the window-max.
- **`fetch_window_at` fallback is load-bearing.** In `drain-cycle.ts`, `const at = entry.fetch.events.at(-1)?.id || fetch_window_at` ‚Äî the `fetch_window_at` fallback is what lets an empty-fetch stream advance so settle converges. Changing it to `?? lease.at` (or similar) hangs the notify/scope suites in an infinite settle loop. Not a bug.
- **The framework has no built-in dedup, by design.** See `docs/docs/architecture/concurrency-model.md` ("why no framework-level dedup"). Do not report "the same event could be delivered twice under X" as a bug ‚Äî at-least-once with consumer-side idempotency is the deliberate contract.
- **Reaction backoff is a persisted per-stream schedule** (since #1262). A retry-with-backoff persists `deferred_at = now + delay` via a due-marked `ack` and releases the lease, so every worker honors the window (the store excludes the stream from `claim`), `retry` advances once per real attempt, and a stream blocks after exactly `maxRetries` attempts regardless of worker count. The due-ack carries the climbing `retry` (budget survives the window); an explicit defer passes `retry: -1` (not a failure). Do **not** re-report the old "per-worker, N√ó amplification, floor = max(configured, leaseMillis)" behavior ‚Äî that was the #1262 bug, now fixed. See `docs/docs/concepts/error-handling.md` ¬ß Backoff.
- **`created` is not monotonic with `id`.** Restore preserves source timestamps verbatim, so timestamp order can diverge from insertion order. Time bounds (`created_before`/`created_after`) are pure filters, never id-ordered early-breaks. (This was #1258; the principle stands for any new code that touches `created`.)
- **Fairness is store-internal by design.** The `Store.claim` signature did not change for the #1223/#1252 fairness reserve ‚Äî each adapter infers the `fair = lagging>=2 ? max(1, floor(lagging/4)) : 0` split inline. Do not propose exposing fairness through the claim interface; that reshape was explicitly rejected.
- **Close/autoclose surfaces are days-only.** Never ms/seconds/minutes on a close-facing API. The one-day floor is deliberate ‚Äî close is low-cadence housekeeping. Not a bug.
- **Windowed close prunes, it does not retire.** `close`/`.autocloses` delete only the prefix below the closest safe `__snapshot__`; no tombstone, subscriptions and cache untouched, stream stays live. No qualifying snapshot ‚áí `skipped`, not an error.
- **Some InMemory shortcuts are legitimate.** InMemory is single-process and dev/test-only. `notify` is intentionally unimplemented. Not every InMemory-vs-SQL difference is a divergence ‚Äî check whether the doc-comment declares the behavior best-effort or capability-gated (e.g. `query_streams.source_matches` MAY return a superset) before calling it a bug.

## Confirmed bugs (regression anchors)

Each of these was found red-first and filed. Future waves should confirm they stay fixed and look for siblings.

- **#1257** ‚Äî `InMemoryStore.restore` kept `pii` inline and never wrote/rolled-back `_pii`, so `forget_pii` no-opped on restored events. Fixed: split pii into `_pii` on restore. *Sibling to check:* any other path that writes events bypassing the `commit` pii-split.
- **#1258** ‚Äî InMemory `query` early-`break`ed on `created_before`/`created_after`, dropping matches when created order ‚âÝ id order. Fixed: `continue`, not `break`. *Sibling:* any other loop that assumes `created` tracks `id`.
- **#1261** ‚Äî time-travel `load({ created_before })` returns wrong (empty) state when a snapshot exists after the cutoff; the `with_snaps` resume floor ignores the time bound (all three adapters). *Sibling:* the `created_after` half of the same branch.
- **#1262** ‚Äî backoff "phantom retry": claiming a stream inside its process-local backoff window bumped persisted `retry` with no handler run, silently halving effective `maxRetries`. Fixed by persisting the backoff window to `deferred_at` (a due-marked `ack` carrying the climbing `retry`), so the store excludes the stream from `claim` during the window. This also refined the `ack` due-marker contract: due-ack now persists the lease's `retry` instead of hardcoding `-1`. *Sibling:* any finalize/claim path where `retry` advances without a real attempt.
- **#1263** ‚Äî `InMemoryStore.block()` on an already-blocked stream returns the lease again (spurious duplicate `blocked` event); PG/SQLite return `[]`. *Sibling:* other InMemory mutators missing a state-guard the SQL `WHERE` clause enforces.
- **#1255** ‚Äî correlate-cycle didn't update `entry.lane` when a higher-priority reaction won the priority merge. Fixed. *Sibling:* other per-entry fields that should follow the priority winner.
- **#1277** ‚Äî `IAct.query`/`query_array` (`act.ts:1228`/`:1268`) hand store events straight to the caller with no `pii_gate`, leaking plaintext PII on all three adapters (worst on PG, which decrypts at-rest ciphertext first). The disclosure gate lives only in `load`. Fail-safe fix is default-deny in the orchestrator (query carries no actor). *Sibling:* any other read surface that returns committed events without routing through `pii_gate`.
- **#1278** ‚Äî partial-progress + backoff drops the persisted `deferred_at` window: the drain-cycle ack ternary (`drain-cycle.ts:243-249`) reaches the `at: acked_at` branch whenever `handled > 0`, so the `next_attempt_at` (due) branch is unreachable and the window/retry are lost. The #1262 sibling the invariants anticipated. Recoverable degradation (watermark still correct), medium. *Sibling:* any other finalize path where `due` and `at` are mutually exclusive but both are needed.
- **#1279** ‚Äî `act-notify` `RedisBroker` disposer calls `unsubscribe(channel)` with no listener arg (`redis.ts:70-72`), which node-redis v4 treats as remove-ALL-listeners ‚Äî so disposing one subscriber on a shared broker silences co-subscribers. The existing `brokers.spec.ts` fake masks it (single-listener overwrite semantics). Wakeup-only degradation, durability intact.
- **#1280** ‚Äî tRPC `status_to_trpc_code` (`trpc/index.ts:231-234`) maps `ConcurrencyError`‚Üí409 and `StreamClosedError`‚Üí412, diverging from Hono/OpenAPI (412/410). Documented-and-tested, so a deliberate-but-inconsistent mapping, not silently-wrong data. Medium. *(Fixed: map by identity ‚Äî Concurrency‚Üí412, StreamClosed‚Üí404 as the one documented tRPC-has-no-410 exception.)*
- **#1286** ‚Äî tRPC generator inlines `options.actor()` in the mutation try (`trpc/index.ts:349`), so an actor "throw to deny" ‚Üí 500, while Hono's `authenticated` returns 401. Breaks the guide's explicit "throw-to-deny is identical / both UNAUTHORIZED" promise. *Sibling:* other per-transport seams (stream/expectedVersion resolver throw) the inline trpc path doesn't map.
- **#1287** ‚Äî OpenAPI marks `Idempotency-Key` `required:false` (`openapi/index.ts:357`) but the route 400s when it's absent (idempotency on). Doc/route mismatch. *Sibling:* the `If-Match` param's emitted optionality vs actual enforcement.
- **#1288** ‚Äî `DeferTimer.schedule` (`defer-timer.ts:101`) clamps waits to ~24.8 days and its comment claims `on_wake` re-arms, but the consumer's `on_wake` (`drain-cycle.ts:373`) only sets `_armed`; the arm-triggered drain early-returns before the `:538` re-schedule. - **#1293** ‚Äî `receiver()` finalizes every delivery twice: the route calls `options.store.commit/release` directly (`receiver/start.ts:114`/`:103`) while the mounted `webhookMiddleware` ALSO auto-finalizes via the `settled`-guarded finalizers (`receiver/hono/index.ts:102-103`).
