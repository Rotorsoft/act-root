# Act Builder

Domain model designer and visualizer for the Act framework. Write Act code in a full-featured editor and see the domain model diagram update in real time.

The builder is a **visualization and design tool**, not a full IDE. It focuses on the domain model — states, actions, events, reactions, projections, and slices — extracting and displaying the structure that matters for understanding and communicating your system design. When importing from GitHub or a local folder, it follows imports from `act()` entry points and collects all reachable source files, preserving the original folder structure. Infrastructure code (servers, databases, UI) is naturally excluded because it's not imported by the domain model.

## Quickstart

```bash
# From the monorepo root
pnpm install
pnpm build   # required for Monaco type definitions

# Optional: configure .env in packages/builder/
cp packages/builder/.env.example packages/builder/.env

pnpm dev:builder
```

This starts both the tRPC server (port 4002) and the Vite client (port 3002).

Open [http://localhost:3002](http://localhost:3002).

### Environment Variables

Create a `.env` file in `packages/builder/`:

| Variable | Required | Description |
|----------|----------|-------------|
| `ANTHROPIC_API_KEY` | For AI generation | Claude API key |
| `GITHUB_TOKEN` or `GH_TOKEN` | For private repos | GitHub token (`gh auth token`) |
| `BUILDER_MODELS` | Optional | Comma-separated model list, format: `id:Label,id:Label` |
| `BUILDER_MAX_TOKENS` | Optional | Default max tokens for AI generation (default: 16384) |
| `BUILDER_SKILLS_DIR` | Optional | Path to Act skill files for AI system prompt |
| `PORT` | Optional | Server port (default: 4002) |
| `VITE_API_URL` | Optional | API URL for the client (default: `http://localhost:4002`) |

## Features

### Project Sources

Open a project from four sources:

- **Todo App** — Built-in sample with states, slices, projections, and orchestrator
- **Open Local** — Browse a local folder using the File System Access API (Chrome/Edge). Recently opened folders are remembered for quick reopen
- **GitHub Import** — Shallow-clone a repo, find `act()` entry points, and follow all imports to collect the domain model files. Supports workspace packages (`@scope/name` → `packages/name/src/`), private repos via `GITHUB_TOKEN`, and preserves the original folder structure
- **AI Generate** — Describe your domain in plain English. Configurable model (Sonnet/Opus/Haiku) and token limit. Streams the response in real time

Recently opened folders and imported repos are shown as quick-access chips in the project dialog.

**How import works:** The builder collects the entire project tree — TypeScript source, `package.json`, `tsconfig.json`, `README.md`, and YAML config files. The file explorer shows the full structure. The domain diagram is extracted only from `.ts/.tsx` files reachable from `act()` entry points. Test directories and build artifacts are excluded.

### VS Code Workbench

Full VS Code experience in the browser:

- **File explorer** — tree view of all project files with folder structure
- **Editor tabs** — open, close, split, and rearrange tabs natively
- **TypeScript IntelliSense** — powered by real `@rotorsoft/act` type definitions
- **Problems panel** — TypeScript errors with click-to-navigate (Ctrl+Shift+M)
- **Search** — find across all project files (Ctrl+Shift+F)
- **Cross-file navigation** — F12 go-to-definition, F2 rename across files
- Click any diagram element to open the file and highlight the definition

### Live Domain Model Diagram

- Auto-generated by evaluating code with mock Act builders
- Visualizes states, actions, events, reactions, projections, and slices
- Color-coded: Actions (blue), Events (orange), States (yellow), Reactions (purple), Projections (green)
- Zoom/pan with mouse wheel + drag
- Clickable elements navigate to code definitions
- Clickable icons on events: projection (green monitor), reaction (purple lightning), guard (red shield) — each with tooltip and code navigation

### AI Refinement

- Inline prompt bar (toggle with the sparkle button in the header) for iterating on loaded code
- Original code stays visible during refinement — applied only on completion
- Error collection: click the error button to append all TypeScript errors to the prompt
- Streaming progress with token count in the header
- Final usage stats (input/output tokens) shown after completion
- System prompt loaded from the repo's Act skill files — single source of truth

### Export

- Download as a **zip archive** preserving the full project folder structure
- For AI-generated projects: includes `package.json` and `tsconfig.json` — ready to unzip, `npm install`, and build
- For imported projects: includes all collected files (source, config, docs) — the zip mirrors the original repo structure
- Package/folder name derived from the project name

## Known Limitations

- **Type acquisition timing** — TypeScript types are loaded via ATA (Automatic Type Acquisition) through an npm registry proxy. First load of a project may take 10-20 seconds for the web tsserver to initialize and fetch types. Subsequent interactions are fast.
- **Workspace packages need `package.json`** — For monorepo workspace imports (`@scope/name`), the builder creates `node_modules` entries automatically from sub-package `package.json` files. If a workspace package doesn't have a `package.json` with a `name` field, its types won't resolve.
- **Private npm packages** — ATA can only fetch types for packages published on the public npm registry. Private/workspace packages (`@risk/domain`) get types through the workspace `node_modules` wiring instead.

## Architecture

```
packages/builder/
├── src/
│   ├── server/
│   │   ├── server.ts           # tRPC server + SSE streaming endpoint
│   │   └── router.ts           # generate, fetchFromGit, config procedures
│   └── client/
│       ├── App.tsx              # Root with tRPC providers
│       ├── Builder.tsx          # Main layout: header, dialog, editor, diagram
│       ├── main.tsx             # Entry point
│       ├── trpc.ts              # tRPC client
│       ├── components/
│       │   ├── Chip.tsx         # Reusable pill with close button
│       │   ├── Tooltip.tsx      # Portal-based tooltip
│       │   ├── CodeEditor.tsx   # Monaco multi-model editor
│       │   ├── Diagram.tsx      # SVG domain model diagram
│       │   ├── GithubIcon.tsx   # GitHub SVG icon
│       │   └── Logo.tsx         # Builder logo
│       ├── lib/
│       │   ├── evaluate.ts      # Mock builder evaluation + model extraction
│       │   ├── mock-builders.ts # Mock Act framework builders
│       │   ├── sort.ts          # Topological sort for file eval order
│       │   ├── validate.ts      # Model validation
│       │   ├── strip-fences.ts  # AI response cleanup utilities
│       │   ├── download.ts      # Shell script project export
│       │   ├── github.ts        # GitHub URL parsing + localStorage history
│       │   ├── local-folder.ts  # File System Access API + IndexedDB history
│       │   └── vscode-init.ts   # VSCode API initialization
│       ├── data/
│       │   ├── prompts.ts       # AI prompt templates
│       │   └── sample-app.ts    # Built-in Todo App sample
│       └── types/
│           ├── index.ts         # Barrel re-exports
│           ├── domain-model.ts  # DomainModel, StateNode, etc.
│           └── file-tab.ts      # FileTab type
├── vite.config.ts               # Vite + workbench + npm registry proxy
└── tsconfig.json
```
